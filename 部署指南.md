# Cloudflare Workers 博客部署指南

## 前置要求

1. 注册 Cloudflare 账号：https://dash.cloudflare.com/sign-up
2. 安装 Node.js（已安装）

---

## 步骤 1: 获取 Cloudflare API Token

1. 登录 https://dash.cloudflare.com
2. 点击右上角头像 → "My Profile"
3. 左侧菜单选择 "API Tokens"
4. 点击 "Create Token"
5. 选择 "Custom token"
6. 配置如下：
   - **Token name**: Blog Deploy
   - **Permissions**:
     - **Account**: Workers Scripts:Edit (或 Workers Scripts:Read + Workers Scripts:Edit)
     - **Account**: Account Settings:Read
     - **User**: Memberships:Read 和 User Details:Read
     - **Zone**: Zone Settings:Read 和 Zone:Read (如果需要自定义域名)
   - **Account Resources**: Include - 你的账号
   - **Zone Resources**: Include - All zones (或特定域名)
   
   💡 **提示**: 如果找不到完全匹配的权限，选择最接近的 Workers 相关权限即可。最低要求是 Workers Scripts 的编辑权限。
7. 点击 "Continue to summary" → "Create Token"
8. **立即复制 Token**（只显示一次！）

---

## 步骤 2: 配置 Wrangler

在项目目录下创建 `.env` 文件：

```bash
CLOUDFLARE_API_TOKEN=你的_token_here
```

或者在命令行设置：

```bash
# Windows PowerShell
$env:CLOUDFLARE_API_TOKEN="你的_token_here"

# Windows CMD
set CLOUDFLARE_API_TOKEN=你的_token_here
```

---

## 步骤 3: 创建 KV 命名空间

运行以下命令：

```bash
# 创建生产环境 KV
npx wrangler kv:namespace create "BLOG_KV"

# 创建预览环境 KV
npx wrangler kv:namespace create "BLOG_KV" --preview
```

执行后会输出类似：
```
🌀 Creating namespace with title "tianliao-blog-BLOG_KV"
✨ Success!
Add the following to your configuration file:
[[kv_namespaces]]
binding = "BLOG_KV"
id = "xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx"
```

**复制 id 值，更新 wrangler.toml 文件**

---

## 步骤 4: 更新 wrangler.toml

```toml
name = "tianliao-blog"
main = "src/index.js"
compatibility_date = "2024-01-01"

[[kv_namespaces]]
binding = "BLOG_KV"
id = "上一步复制的id"
preview_id = "预览环境的id"

[vars]
SITE_TITLE = "天聊博客"
SITE_DESCRIPTION = "一个基于 Cloudflare Workers 的博客"
ADMIN_USERNAME = "admin"
```

---

## 步骤 5: 设置管理员密码

```bash
npx wrangler secret put ADMIN_PASSWORD
```

输入你的管理员密码（输入时不会显示）

---

## 步骤 6: 本地测试（可选）

```bash
npm run dev
```

访问 http://localhost:8787 查看效果

---

## 步骤 7: 部署

```bash
npm run deploy
```

部署成功后会显示：
```
✨ Successfully published
  https://tianliao-blog.your-subdomain.workers.dev
```

## 步骤 8: 初始化数据

### 8.1 创建分类

```bash
npx wrangler kv:key put --binding=BLOG_KV "categories" "[{\"id\":\"1\",\"name\":\"技术\",\"slug\":\"tech\",\"description\":\"技术文章\",\"createdAt\":\"2024-01-01T00:00:00Z\"},{\"id\":\"2\",\"name\":\"生活\",\"slug\":\"life\",\"description\":\"生活随笔\",\"createdAt\":\"2024-01-01T00:00:00Z\"}]"
```

### 8.2 创建示例文章

```bash
npx wrangler kv:key put --binding=BLOG_KV "posts" "[{\"id\":\"1\",\"title\":\"欢迎使用天聊博客\",\"slug\":\"welcome\",\"content\":\"这是一篇示例文章，支持 **Markdown** 格式。\\n\\n## 特性\\n\\n- 快速\\n- 免费\\n- 易用\",\"category\":\"tech\",\"tags\":[\"intro\",\"welcome\"],\"status\":\"published\",\"views\":0,\"createdAt\":\"2024-01-01T00:00:00Z\",\"updatedAt\":\"2024-01-01T00:00:00Z\"}]"
```

## 常见问题

### 1. 部署失败 "Authentication error"
- 检查 API Token 是否正确
- 检查 Token 权限是否包含 Workers:Edit

### 2. KV 数据读取失败
- 确认 wrangler.toml 中的 id 和 preview_id 正确
- 确认 KV namespace 已创建

### 3. 登录失败
- 确认已设置 ADMIN_PASSWORD
- 确认用户名和密码正确

### 4. 页面显示 "Not Found"
- 确认已初始化 posts 和 categories 数据

---

## 自定义域名（可选）

1. 在 Cloudflare Dashboard 添加你的域名
2. 进入 Workers & Pages
3. 选择你的 Worker
4. 点击 "Triggers" → "Custom Domains"
5. 添加域名或子域名（如 blog.yourdomain.com）

---

## 更新博客

修改代码后重新部署：

```bash
npm run deploy
```

数据会保留在 KV 中，不会丢失。
